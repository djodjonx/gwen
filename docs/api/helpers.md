# API Helpers

GWEN provides several `define*` helpers for creating game elements.

Each helper (except `defineConfig`) supports **two syntaxes**:

| Syntax | When to use |
|--------|-------------|
| **Form 1 ‚Äî direct object** | Simple case, no local state needed |
| **Form 2 ‚Äî factory** (`name, () => ‚Ä¶`) | Local state in closure, or dynamic setup |

---

## defineComponent

Define ECS components.

**Form 1 ‚Äî direct object** (recommended):
```typescript
import { defineComponent, Types } from '@gwen/engine-core';

export const Position = defineComponent({
  name: 'position',
  schema: { x: Types.f32, y: Types.f32 }
});
```

**Form 2 ‚Äî factory** (dynamic schema or local constants):
```typescript
export const Position = defineComponent('position', () => ({
  schema: { x: Types.f32, y: Types.f32 }
}));
```

[Learn more](/core/components)

## defineScene

Define game scenes.

**Form 1 ‚Äî direct object** (no external dependencies):
```typescript
import { defineScene } from '@gwen/engine-core';

export const PauseScene = defineScene({
  name: 'Pause',
  systems: [],
  ui: [],
  onEnter(api) { },
  onExit(api) { }
});
```

**Form 2 ‚Äî factory** (typed dependencies injected at call site):
```typescript
import type { EngineAPI } from '@gwen/engine-core';

export const GameScene = defineScene('Game', () => ({
  systems: [MovementSystem, PlayerSystem],
  ui: [PlayerUI, ScoreUI],
  onEnter(api: EngineAPI<GwenServices>) {
    api.prefabs.register(PlayerPrefab);
    api.prefabs.instantiate('Player');
    api.services.get('audio').play('bgm'); // ‚úÖ Typed
  },
  onExit(api: EngineAPI<GwenServices>) { }
}));
```

> `EngineAPI<GwenServices>` is optional ‚Äî omit it if you don't call `api.services`. The annotation applies per-method.
> `GwenServices` is a global type generated by `gwen prepare` ‚Äî no import needed.

> With Form 2, `defineScene` returns a **callable factory** ‚Äî call it to get the `Scene` object:
> ```typescript
> scenes.register(GameScene); // GameScene is already a Scene (factory called with no args)
> ```

[Learn more](/core/scenes)

## definePrefab

Define reusable entity templates.

**Form 1 ‚Äî direct object**:
```typescript
import { definePrefab } from '@gwen/engine-core';

export const PlayerPrefab = definePrefab({
  name: 'Player',
  create: (api) => {
    const id = api.createEntity();
    api.addComponent(id, Position, { x: 240, y: 560 });
    return id;
  }
});
```

**Form 2 ‚Äî factory** (local constants or shared state in closure):
```typescript
export const EnemyPrefab = definePrefab('Enemy', () => {
  const baseSpeed = 80; // captured once

  return {
    create: (api, x: number, y: number) => {
      const id = api.createEntity();
      api.addComponent(id, Position, { x, y });
      api.addComponent(id, Velocity, { vx: 0, vy: baseSpeed });
      return id;
    }
  };
});
```

[Learn more](/core/prefabs)

## defineUI

Define renderer-agnostic UI components.

The optional generic `<Services>` types `api.services.get()` inside `render`, `onMount` and `onUnmount`.
Without it, `api.services.get(...)` returns `unknown` ‚Äî which is fine if you don't call any service, or if you cast manually.

**`GwenServices` is a global type** generated by `gwen prepare` ‚Äî no import needed.

```typescript
// Without generic ‚Äî api.services is untyped
export const BulletUI = defineUI({
  name: 'BulletUI',
  render(api, id) {
    // api.services.get('renderer') ‚Üí unknown ‚öÝÔ∏è
  }
});

// With GwenServices ‚Äî api.services is fully typed (no import needed)
export const BulletUI = defineUI<GwenServices>({
  name: 'BulletUI',
  render(api, id) {
    const { ctx } = api.services.get('renderer'); // ‚Üí Canvas2DRenderer ‚úÖ
  }
});
```

**Form 1 ‚Äî direct object** (stateless rendering):
```typescript
import { defineUI } from '@gwen/engine-core';

export const BulletUI = defineUI<GwenServices>({
  name: 'BulletUI',
  render(api, id) {
    const pos = api.getComponent(id, Position);
    if (!pos) return;
    const { ctx } = api.services.get('renderer');
    ctx.fillStyle = '#fff';
    ctx.fillRect(pos.x - 2, pos.y - 6, 4, 12);
  }
});
```

**Form 2 ‚Äî factory** (local state in closure, e.g. animation counter):
```typescript
export const PlayerUI = defineUI<GwenServices>('PlayerUI', () => {
  // Local state ‚Äî not shared between entities
  let animFrame = 0;

  return {
    onMount(api, id) {
      animFrame = 0;
    },
    render(api, id) {
      animFrame++;
      const pos = api.getComponent(id, Position);
      if (!pos) return;
      const { ctx } = api.services.get('renderer');
      ctx.fillStyle = '#4fffb0';
      ctx.fillRect(pos.x - 16, pos.y - 16, 32, 32);
    },
    onUnmount(api, id) { }
  };
});
```

> `<GwenServices>` is the same global type used by `defineSystem`, `defineScene` and `definePrefab`. See [GwenServices ‚Äî the global type](#gwenservices--the-global-type).

[Learn more](/core/ui)

## defineSystem

Define game systems (pure gameplay logic).

**Form 1 ‚Äî direct object** (no local state):
```typescript
import { defineSystem } from '@gwen/engine-core';
import type { EngineAPI } from '@gwen/engine-core';

export const MovementSystem = defineSystem({
  name: 'MovementSystem',
  onUpdate(api: EngineAPI<GwenServices>, dt) {
    const entities = api.query([Position, Velocity]);
    for (const id of entities) {
      const pos = api.getComponent(id, Position);
      const vel = api.getComponent(id, Velocity);
      if (!pos || !vel) continue;
      api.addComponent(id, Position, {
        x: pos.x + vel.vx * dt,
        y: pos.y + vel.vy * dt,
      });
    }
  }
});
```

> `EngineAPI<GwenServices>` is optional ‚Äî omit it if you don't call `api.services`. Without it, `api.services.get(...)` returns `unknown`.
> `GwenServices` is a global type generated by `gwen prepare` ‚Äî no import needed.

**Form 2 ‚Äî factory** (local state in closure, avoids global variables):
```typescript
export const SpawnerSystem = defineSystem('SpawnerSystem', () => {
  let timer = 0; // private state

  return {
    onInit() { timer = 0; },
    onUpdate(api, dt) {
      timer += dt;
      if (timer >= 2.0) {
        timer = 0;
        api.prefabs.instantiate('Enemy', Math.random() * 800, 0);
      }
    }
  };
});

// Registration ‚Äî call the factory to get an instance
export const GameScene = defineScene('Game', () => ({
  systems: [MovementSystem, SpawnerSystem()], // SpawnerSystem() ‚Üê factory call
  onEnter(api) { },
  onExit(api) { }
}));
```

Systems are for game mechanics, entity processing, and state management.

## defineConfig

Configure the engine and register plugins.

> `defineConfig` has only **one form** ‚Äî a direct configuration object.
> The service types are automatically inferred from `tsPlugins`.

```typescript
import { defineConfig } from '@gwen/engine-core';

export const gwenConfig = defineConfig({
  engine: { maxEntities: 5000, targetFPS: 60, debug: false },
  html: { title: 'My Game', background: '#000000' },
  tsPlugins: [
    new InputPlugin(),
    new AudioPlugin({ masterVolume: 0.8 }),
    new Canvas2DRenderer({ width: 800, height: 600 })
  ]
});
```

The generic parameter `Services` is automatically inferred from the declared `tsPlugins`, enabling type-safe `api.services.get()` calls everywhere.

[Learn more](/core/configuration)

## GwenServices ‚Äî the global type

`GwenServices` is a **global TypeScript type** automatically generated by `gwen prepare` (called automatically by `gwen dev` and `gwen build`).

It is inferred from the `tsPlugins` declared in your `gwen.config.ts` and written into `.gwen/gwen.d.ts` ‚Äî **no export, no import required anywhere in your project**.

```typescript
// gwen.config.ts ‚Äî just declare your plugins
import { defineConfig } from '@gwen/engine-core';
import { InputPlugin } from '@gwen/plugin-input';
import { AudioPlugin } from '@gwen/plugin-audio';

export const gwenConfig = defineConfig({
  tsPlugins: [new InputPlugin(), new AudioPlugin({ masterVolume: 0.8 })],
});
```

```typescript
// systems/PlayerSystem.ts ‚Äî GwenServices available globally, no import
import type { EngineAPI } from '@gwen/engine-core';

export const PlayerSystem = defineSystem({
  name: 'PlayerSystem',
  onUpdate(api: EngineAPI<GwenServices>, dt) {
    const kb = api.services.get('keyboard'); // ‚úÖ Fully typed
  }
});
```

The generated `.gwen/gwen.d.ts` looks like:

```typescript
// .gwen/gwen.d.ts ‚Äî auto-generated by `gwen prepare`, do not edit
import type { GwenConfigServices } from '@gwen/engine-core';
import type _cfg from '../gwen.config';

declare global {
  type GwenServices = GwenConfigServices<typeof _cfg>;
  // ‚Üí { keyboard: KeyboardInput; audio: AudioManager; ‚Ä¶ }
}
```

> Run `gwen prepare` once after adding or removing plugins to refresh the type.

## GwenConfigServices

`GwenConfigServices<T>` is the utility type used internally by `gwen prepare` to extract the merged services map from a `defineConfig()` result. You can also use it directly if you need the services type as a named export (e.g. for a shared library or a custom plugin):

```typescript
import { defineConfig, GwenConfigServices } from '@gwen/engine-core';
import { InputPlugin } from '@gwen/plugin-input';

export const gwenConfig = defineConfig({
  tsPlugins: [new InputPlugin()],
});

// Manual extraction ‚Äî useful for library authors or advanced use cases
export type MyProjectServices = GwenConfigServices<typeof gwenConfig>;
// ‚Üí { keyboard: KeyboardInput; mouse: MouseInput; gamepad: GamepadInput }
```

For normal game development, rely on the global `GwenServices` generated by `gwen prepare` instead.

## GwenPlugin Interface

Create plugins by implementing `GwenPlugin<N, P>`:

```typescript
import type { GwenPlugin, EngineAPI } from '@gwen/engine-core';

export class MyPlugin implements GwenPlugin<'MyPlugin', { myService: MyService }> {
  readonly name = 'MyPlugin' as const;
  readonly provides = { myService: {} as MyService };

  onInit(api: EngineAPI) {
    api.services.register('myService', { /* ... */ });
  }
}
```

## Quick Comparison

| Helper | Form 1 (object) | Form 2 (factory) | Local state | `<Services>` generic |
|--------|----------------|------------------|-------------|----------------------|
| `defineComponent` | ‚úÖ `{ name, schema }` | ‚úÖ `('name', () => ({ schema }))` | ‚Äî | ‚úó no `api` |
| `defineScene` | ‚úÖ `{ name, onEnter, onExit }` | ‚úÖ `('name', () => ({ onEnter, onExit }))` | ‚Äî | ‚úÖ on `api` param |
| `definePrefab` | ‚úÖ `{ name, create }` | ‚úÖ `('name', () => ({ create }))` | ‚úÖ closure | ‚úÖ on `api` param |
| `defineUI` | ‚úÖ `{ name, render }` | ‚úÖ `('name', () => ({ render }))` | ‚úÖ closure | ‚úÖ on the helper itself `defineUI<GwenServices>(...)` |
| `defineSystem` | ‚úÖ `{ name, onUpdate }` | ‚úÖ `('name', () => ({ onUpdate }))` | ‚úÖ closure | ‚úÖ on `api` param |
| `defineConfig` | ‚úÖ `{ engine, tsPlugins }` | ‚úó not applicable | ‚Äî | ‚úó inferred automatically |

**`<Services>` quick rule:**
- `defineUI` ‚Üí put the generic on the helper: `defineUI<GwenServices>(...)`
- `defineSystem`, `defineScene`, `definePrefab` ‚Üí annotate the `api` parameter: `(api: EngineAPI<GwenServices>)`
- `defineComponent`, `defineConfig` ‚Üí no generic needed
